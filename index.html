II Execution contexts and lexical environments

5 syntax parsers execution contexts lexical environments

A syntax parser:

  • reads your code (character by character)
  • determines what it does and if its grammar is valid

lexical environments: exists in programming languages in which where you write something is important.

execution contexts: manages which lexical environment is currently running. It can contain things beyond what is written in your code.

6 name value pairs and objects

Name/value pairs - A name which maps to a unique value

Object - A collection of name value pairs

Address: { Street: 'Main', Number: 100, Apartment: { Floor: 3, Number: 301 } }

Address is the name. The value is a collection of other name/value pairs.

7 The Global Environment and the Global Object

JavaScript code runs inside an execution context. Lets look at an execution context being created and run. Normally more than 1 execution context runs during execution of a JavaScript program. The base context is `Global`. It is accessible everywhere to everything in your program./p>

The Global execution context creates 2 things for you:

  • Global Object
  • the `this` variable

Even with an empty app.js file an execution context is created. At the global level 

this
refers to the window object. In JavaScript 'global' means not inside a function.
